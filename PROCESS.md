# ขั้นตอนการทำงานของโปรเจ็กต์ Daily Price Scraper

เอกสารนี้อธิบายกระบวนการและภาพรวมการทำงานทั้งหมดของระบบดึงข้อมูลราคาสินค้าเกษตรและพลังงานจาก CME Group ในแต่ละวัน

## 1. จุดประสงค์ (Objective)
เพื่อดึงข้อมูลราคาเทรด (Settlements) สินค้าล่วงหน้าจากเว็บไซต์ CME Group จำนวน 10 ประเภทสินค้า โดยอัตโนมัติเป็นประจำทุกวันเวลา 12:30 น. (ตามเวลาประเทศไทย) และบันทึกลงในไฟล์ CSV เพียงไฟล์เดียวเพื่อเป็นฐานข้อมูลสำหรับการวิเคราะห์หรือใช้งานต่อไป

## 2. โครงสร้างไฟล์และส่วนประกอบ (Files & Components)
- `scraper.py`: ไฟล์สคริปต์หลัก (Main script) ที่ใช้ควบคุมการดึงข้อมูลและระบบตั้งเวลา (Scheduler) 
- `products.json`: ไฟล์กำหนดค่า (Configuration) สำหรับสินค้า 10 ตัว ประกอบด้วยชื่อสินค้า (Name), รหัสสินค้าทาง API (Product ID) และชื่อสั้นๆ สำหรับระบุตัวชี้ (Slug)
- `requirements.txt`: รายชื่อไลบรารีของ Python ที่จำเป็นต้องใช้ (เช่น `playwright`, `schedule`)
- `data/cme_daily_prices.csv`: ไฟล์ Output ที่จะบันทึกข้อมูลทุกวันโดยเพิ่มข้อมูลต่อท้ายบรรทัดเดิมไปเรื่อยๆ (Append mode)

## 3. ขั้นตอนการทำงานของโปรแกรม (How it works)

โปรแกรมถูกออกแบบมาเพื่อหลีกเลี่ยงระบบป้องกันบอทของเว็บไซต์ CME Group โดยมีลำดับขั้นตอนดังนี้:

**3.1 การเตรียมเบราว์เซอร์ (Browser Initialization)**
- โปรแกรมใช้ไลบรารี `Playwright` เพื่อจำลองการเปิดเบราว์เซอร์
- ใช้เอนจินของ **Microsoft Edge** (`channel="msedge"`) เพื่อทำให้เหมือนผู้ใช้งานทั่วไป และซ่อนสถานะว่าเป็นการใช้บอท (Bypass Bot Detection)

**3.2 ตรวจสอบข้อมูลแบบวนลูปทีละสินค้า (Iterate Products)**
- โปรแกรมจะอ่านรายชื่อสินค้าทั้งหมดจากไฟล์ `products.json` แล้ววนลูปดึงข้อมูลทีละตัว

**3.3 ค้นหาวันที่เทรดล่าสุด (Fetch Latest Trade Date)**
- โปรแกรมจะยิง Request ในรูปแบบ API ไปยัง CME Group เพื่อหาวันที่อัปเดตล่าสุดของแต่ละสินค้า (`/TradeDate/{product_id}`)

**3.4 ดึงข้อมูลราคา (Fetch Settlement Data)**
- เมื่อได้วันที่ล่าสุดแล้ว โปรแกรมจะยิง Request อีกครั้งไปเรียก API ข้อมูลราคา (`/Settlements/{product_id}/...`)
- ข้อมูลได้ที่ตอบกลับมาจะอยู่ในรูปแบบ JSON ซึ่งจะใช้งานได้ง่ายและแม่นยำกว่าการสแกนหน้าเว็บ (DOM Scraping) ที่ต้องรอให้เว็บไซต์โหลดเสร็จสมบูรณ์ 

**3.5 รวบรวมข้อมูลและแปลงโครงสร้าง (Extract and Transform)**
- โปรแกรมจะดึงข้อมูลแรก (แถวบนสุด - แถวที่บอกช่วงเดือนทำการล่าสุด) 
- ดึงข้อมูล 9 คอลัมน์ที่ต้องการ ได้แก่: Month, Open, High, Low, Last, Change, Settle, Est. Volume, Prior Day OI
- เพิ่มวันที่ (Date) และชนิดสินค้า (Product) ไว้ด้านหน้าของ 9 คอลัมน์ที่ดึงมาเพื่อให้รู้ว่าแถวนั้นเป็นข้อมูลของวันไหนและของสินค้าอะไร

**3.6 บันทึกผลลัพธ์ลง CSV (Save to Master CSV)**
- นำข้อมูลรวมทั้งหมดไปเขียนต่อท้ายลงที่ไฟล์ `data/cme_daily_prices.csv` ทำซ้ำไปจนครบ 10 ผลิตภัณฑ์

## 4. วิธีการใช้งานโปรแกรมตระกูลดึงข้อมูล (Usage)

**การรันแบบทำทันที 1 ครั้ง (One-time Scrape):**
```bash
python scraper.py
```
*(เมื่อสั่งงานเสร็จโปรแกรมจะถูกปิดอัตโนมัติ)*

**การรันแบบตั้งเวลาเป็นรอบแบบทำทุกวัน (Cron mode):**
```bash
python scraper.py --cron
```
*(โปรแกรมจะรันค้างไว้เบื้องหลัง และจะทำงานอัตโนมัติในทุกๆ วันเวลา 12:30 น.)*

## 5. กระบวนการเพิ่มสินค้าในอนาคต (Adding New Products)
หากในอนาคตต้องการเพิ่มสินค้าใหม่ที่หน้าตาเว็บไซต์เป็นแบบเดิม ให้ใช้คำสั่งสำหรับหารหัส Product ID แบบอัตโนมัติ:
```bash
python scraper.py --discover <URL_ของสินค้าจากเว็บไซต์_CME>
```
โปรแกรมจะเปิดเบราว์เซอร์เข้าไปดูและสกัดหารหัสสินค้า แล้วปริ้นท์สิ่งที่ต้องนำไปใส่ใน `products.json` ออกมาให้โดยอัตโนมัติ
